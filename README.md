# College Project — Multi-Page Web Application

A responsive, multi-page web application built from scratch using **HTML**, **CSS**, and **vanilla JavaScript**. Designed as a hands-on exercise in front-end fundamentals: layout systems, DOM manipulation, and cross-browser compatibility.

## Features

- **Responsive layout** — works across mobile, tablet, and desktop viewports
- **Interactive UI** — JavaScript-driven navigation, form validation, and dynamic content rendering
- **Semantic HTML** — proper use of `<header>`, `<main>`, `<section>`, `<nav>`, and `<footer>` elements
- **Custom CSS** — no utility-class frameworks; all styling is hand-written for full control

## Tech Stack

| Layer | Technology |
|-------|-----------|
| Structure | HTML5 |
| Styling | CSS3 (Flexbox, Grid, media queries) |
| Interactivity | Vanilla JavaScript (ES6+) |

## Getting Started

```bash
# Clone the repository
git clone https://github.com/11evan24/College-Project.git

# Open in browser
cd College-Project
# Open index.html in your browser — no build step required
```

## Project Structure

```
College-Project/
├── index.html          # Entry point
├── css/
│   └── style.css       # Global styles
├── js/
│   └── script.js       # DOM logic and event handlers
└── assets/
    └── images/         # Static assets
```

## What I learned

- How CSS Flexbox and Grid differ in layout behaviour
- Writing accessible, semantic HTML that doesn't rely on `<div>` soup
- Handling browser inconsistencies without a polyfill library
- Structuring vanilla JS for readability without a framework's opinions
